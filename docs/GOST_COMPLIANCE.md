# Документация по соответствию ГОСТ Р ИСО/МЭК 27001-2021

**Дата**: 28 октября 2025 г.
**Версия**: 1.0
**Статус**: Утверждено

## 1. Введение

Настоящий документ описывает соответствие платформы A2D2 требованиям ГОСТ Р ИСО/МЭК 27001-2021 «Информационная безопасность, кибербезопасность и защита частной жизни. Системы менеджмента информационной безопасности. Требования».

## 2. Область применения СМИБ

### 2.1 Охват системы менеджмента

Система менеджмента информационной безопасности (СМИБ) охватывает:
- Веб-приложение платформы A2D2
- Базу данных PostgreSQL
- Хранилище файлов
- API интерфейсы
- Процессы обработки персональных данных
- Инфраструктуру хостинга

### 2.2 Заинтересованные стороны

- Пользователи платформы
- Администраторы системы
- Операторы роботизированных систем
- Руководство организации
- Регуляторные органы (Роскомнадзор, ФСТЭК)

## 3. Контекст организации (Раздел 4 ГОСТ)

### 3.1 Понимание организации и ее контекста

Платформа A2D2 предназначена для:
- Автоматизации бизнес-процессов с применением ИИ
- Управления роботизированными системами
- Обработки и анализа данных

**Внешние факторы**:
- Требования законодательства РФ (ФЗ-152, ФЗ-149)
- Киберугрозы и атаки
- Технологические изменения

**Внутренние факторы**:
- Культура информационной безопасности
- Доступные ресурсы
- Структура организации

### 3.2 Требования заинтересованных сторон

| Сторона | Требования | Реализация |
|---------|-----------|-----------|
| Пользователи | Конфиденциальность данных | Шифрование, контроль доступа |
| Руководство | Непрерывность бизнеса | Резервное копирование, мониторинг |
| Регуляторы | Соответствие ФЗ-152 | Политика конфиденциальности, аудит |

### 3.3 Область применения СМИБ

**Включено**: Все компоненты платформы A2D2, процессы обработки данных, персонал.

**Исключено**: Физическая безопасность помещений (ответственность ЦОД), конечные устройства пользователей.

## 5. Планирование (Раздел 6 ГОСТ)

### 5.1 Оценка рисков

#### 5.1.1 Идентифицированные риски

| Риск | Вероятность | Влияние | Оценка | Меры контроля |
|------|-------------|---------|--------|---------------|
| Несанкционированный доступ | Средняя | Высокое | Высокий | MFA, RBAC, мониторинг |
| Утечка ПДн | Низкая | Критическое | Высокий | Шифрование, аудит, DLP |
| DDoS атака | Средняя | Среднее | Средний | Rate limiting, WAF |
| Компрометация учетных данных | Средняя | Высокое | Высокий | Сложные пароли, MFA |
| Уязвимости ПО | Высокая | Среднее | Высокий | Patch management, scanning |
| Потеря данных | Низкая | Критическое | Средний | Резервное копирование |

#### 5.1.2 Критерии принятия рисков

- **Приемлемый риск**: Низкий
- **Требуется дополнительный контроль**: Средний
- **Неприемлемый риск**: Высокий, Критический

### 5.2 Цели информационной безопасности

1. **Конфиденциальность**: Обеспечить защиту персональных данных в соответствии с ФЗ-152
   - Метрика: 0 инцидентов утечки данных

2. **Целостность**: Обеспечить точность и неизменность критичных данных
   - Метрика: 100% логов с валидными контрольными суммами

3. **Доступность**: Обеспечить uptime 99.5%
   - Метрика: Измерение через мониторинг

4. **Соответствие**: Соблюдение всех применимых требований законодательства
   - Метрика: 0 нарушений по результатам аудита

## 8. Оценка результатов деятельности (Раздел 9 ГОСТ)

#### 8.1.1 Метрики безопасности

| Метрика | Целевое значение | Частота измерения |
|---------|-----------------|-------------------|
| Uptime | ≥99.5% | Ежедневно |
| Инциденты безопасности | 0 критичных | Ежемесячно |
